# ADMIN GOVERNANCE MODE — Definition & Activation Rules

Version: 1.0
Status: ACTIVE AFTER CORE LOCK
Applies from: FAZA 60
Authority: System Governance Framework

---

## 1. NAMEN

Ta dokument definira ADMIN GOVERNANCE MODE kot edini dovoljeni
upravljalski kontekst za potrjevanje politik, distribucijskih profilov
in drugih upravljalskih odločitev po zaklepu jedra.

---

## 2. DEFINICIJA MODE-A

ADMIN GOVERNANCE MODE ni uporabniški vmesnik, okno ali aplikacija.

Je:
- upravljalsko stanje sistema,
- v katerem so dovoljene odločitve z dolgoročnimi posledicami,
- in v katerem je zahtevana eksplicitna avtoriteta.

---

## 3. AKTIVACIJA

ADMIN GOVERNANCE MODE se:
- nikoli ne aktivira implicitno,
- nikoli ne aktivira z navadnim pogovorom,
- vedno zahteva zavestno in eksplicitno dejanje.

Deklaracija vloge v chatu (npr. "kot ADMIN") je zahteva po kontekstu,
ne pa avtomatska podelitev pravice.

---

## 4. RAZMERJE DO CHATA

Chat:
- lahko pripravlja osnutke,
- lahko razlaga politike,
- lahko simulira posledice.

Chat:
- nima izvršilne moči,
- ne more zapisovati aktivnih politik,
- ne more potrjevati sprememb.

Vsaka chat-generirana politika je neveljavna,
dokler ni potrjena v ADMIN GOVERNANCE MODE.

---

## 5. POTRDITVENI AKT

Veljavna upravljalska odločitev zahteva:
- ločeno potrditev,
- preverjeno vlogo,
- revizijski zapis.

Primer:
"Potrjujem kot ADMIN" je veljaven akt le,
če sistem potrdi obstoj ADMIN GOVERNANCE MODE.

---

## 6. REVIZIJA IN SLEDLJIVOST

Vsak vstop v ADMIN GOVERNANCE MODE in vsaka potrjena sprememba:
- se zabeleži,
- je časovno sledljiva,
- ima odgovorno osebo.

---

## 7. KONČNA DOLOČBA

ADMIN GOVERNANCE MODE zagotavlja,
da sistem ostane upravljan, a ne zlorabljiv,
ter da chat ostane vmesnik, ne vir moči.

END OF DOCUMENT
